codex-auto Python UI(Tkinter) 기능 상세 정리
=========================================

개요
----
- codex-auto의 데스크톱 UI는 Tkinter 기반 2-스테이지 화면 구조(설정 화면 + 실행 플로우 화면)로 동작한다.
- 목적은 "여러 저장소를 워크스페이스에서 관리"하고 "Codex 기반 실행 계획(Plan) 생성/편집/실행/마감(closeout)"까지 한 화면 흐름에서 처리하는 것이다.

실행/진입점
----------
- GUI 실행 방법
  - `codex-auto-gui`
  - `python -m codex_auto.gui`
  - 패키지 설치 전에는 `python gui_main.py`
- 메인 엔트리포인트에서 `Tk()` 루트를 생성하고 `CodexAutoGUI`를 초기화한 뒤 `mainloop()`를 시작한다.

워크스페이스/프로젝트 관리
------------------------
- GUI 워크스페이스 경로 결정 우선순위
  1) `CODEX_AUTO_GUI_WORKSPACE` 환경변수
  2) 현재 경로의 `.codex-auto-workspace` (존재 시)
  3) 홈 디렉터리의 `.codex-auto-workspace`
- 프로젝트 목록(Managed Projects)
  - 등록된 프로젝트를 카드형 목록으로 표시
  - 카드에는 표시 이름, 현재 상태, 로컬 경로, 원격 URL/브랜치 정보가 보임
  - 선택된 카드는 강조 스타일 적용
- 프로젝트 액션
  - Create New: 신규 관리 프로젝트 등록 폼 열기
  - Edit Selected: 선택 프로젝트를 설정 폼에 로딩
  - Refresh: 워크스페이스의 프로젝트 목록/상태 다시 읽기
- 프로젝트 요약 패널
  - 현재 프로젝트의 핵심 상태를 텍스트로 요약
  - 남은 스텝, closeout 상태, 최근 블록 상태, 마지막 실행 시각 등을 표시

2단계 화면 구조
--------------
1) Setup Stage(설정 단계)
- 왼쪽: 프로젝트 목록 + 요약
- 오른쪽: 초기 안내 또는 프로젝트 생성/수정 폼
- 폼 입력 항목
  - Project Name(표시 이름)
  - Working Directory(로컬 저장소 경로, Browse 지원)
  - GitHub Connection 모드
    - existing: 폴더의 기존 origin 사용
    - manual: URL 직접 입력(이 모드일 때만 URL 입력 필드 노출)
    - none: 아직 GitHub 연결 안 함
  - Verification Command(기본 테스트/검증 명령)
  - Execution Model(검증된 모델 프리셋 선택)
- 폼 액션
  - Save/Create: 프로젝트 설정 저장
  - Next: 저장 후 플로우 화면으로 이동
  - Cancel: 설정 폼 닫고 안내 패널로 복귀

2) Flow Stage(실행 단계)
- 상단 Prompt & Plan 영역
  - 목표 프롬프트 입력(자연어)
  - 버튼
    - Generate Plan With Codex: 프롬프트 기반 실행 계획 생성
    - Save Edited Plan: UI에서 수정한 계획 저장
    - Reset Plan: 저장된 프롬프트/스텝 초기화
    - Run Remaining Steps: 미완료 스텝 순차 실행
    - Run Closeout: 전체 스텝 완료 후 최종 마감 실행
    - Stop After Current Step: 현재 스텝 완료 후 일시 정지 요청
    - Reload Project: 디스크의 최신 상태 재로딩
  - 상태 배지(status pill)로 현재 작업 상태 표시
- 좌측 Interactive Flow
  - 스텝 + closeout 노드를 캔버스에서 시각화
  - 노드 상태별 색상(예: pending/completed/running/failed/paused)
  - 선택 노드 강조
  - 노드 간 화살표로 실행 순서 표시
- 우측 Selected Step 편집기
  - 선택 스텝의 상세 편집
    - Title
    - Test Command
    - Display Description
    - Codex Instruction
    - Success Criteria
  - 편집 액션
    - Save Step
    - Add Step(선택 스텝 뒤 또는 끝에 추가)
    - Delete Step
    - Move Up/Down
    - Clear Selection
  - 완료(completed) 스텝은 읽기 전용이며 pending 스텝만 편집 가능

모델/런타임 설정 동작
-------------------
- 모델은 미리 정의된 프리셋 목록 중심으로 선택한다.
- 과거에 저장된 커스텀 모델이 있으면 "Saved Custom Model" 항목을 별도 노출한다.
- 런타임 요약 문자열로 모델/추론 강도 정보를 화면에 즉시 반영한다.
- 실행 시 UI가 구성하는 핵심 런타임 기본값
  - approval_mode: `never`
  - sandbox_mode: `danger-full-access`
  - allow_push: True
  - checkpoint_interval_blocks: 1
  - max_blocks: 화면 입력값(최소 1)

비동기 실행/응답성
-----------------
- 오래 걸리는 작업(목록 갱신, 계획 생성/저장, 스텝 실행, closeout)은 백그라운드 스레드에서 수행한다.
- 메인 UI 스레드는 queue polling(`after`)으로 이벤트를 받아 화면을 갱신한다.
- 동시에 하나의 백그라운드 작업만 허용하여 충돌을 방지한다(`busy` 플래그).

로그/스냅샷/상태 표시
-------------------
- Activity 로그
  - 시작/완료/오류/수정 이력을 누적 표시
  - 내부 버퍼는 최근 1000줄 유지
- Snapshot
  - 프로젝트/런타임/플랜 상태를 JSON 형태로 보여줌
  - 프로젝트 목록 통계 스냅샷도 제공(ready/running/failed 집계)
- 상태 텍스트
  - "Ready", "... in progress", 실행 중 스텝 식별자 등 실시간 반영

플랜 편집/검증 규칙
------------------
- 프롬프트만 있고 스텝이 비어 있으면 저장/실행 전 에러 처리한다.
- 선택 스텝 제목이 비어 있으면 저장 불가.
- 스텝 재생성/리셋 시 사용자 확인 다이얼로그를 사용한다.
- 완료된 스텝이 포함된 계획은 재생성(generate) 제한(실수로 이력 손실 방지).
- 스텝 이동은 pending 구간 내부에서만 허용.

실행(run) 흐름
-------------
- Run Remaining Steps
  1) 현재 편집본을 먼저 저장(update_execution_plan)
  2) 미완료 스텝을 앞에서부터 순차 실행
  3) 각 스텝 실행 전 "running" 표시, 실행 후 결과 상태 반영
  4) 실패/중단 시 최신 프로젝트 상태를 다시 로딩하여 UI 일관성 유지
- Stop After Current Step
  - 즉시 중단이 아니라 "현재 스텝 종료 후 다음 스텝 진입 전 멈춤" 방식
- Run Closeout
  - 모든 스텝 completed 상태일 때만 실행 가능
  - 최종 정리/검증/문서 핸드오프용 마감 절차 수행

사용성(UI/디자인)
----------------
- 사용자 정의 테마/색상/컴포넌트 스타일(ttk Style) 적용
- 스크롤 가능한 프로젝트 카드 목록과 플로우 캔버스 제공
- 텍스트 입력 영역(프롬프트/설명/지시문/성공 조건/로그/스냅샷)에 통일된 스타일 적용
- 단계 전환(Setup ↔ Flow) 버튼 기반 네비게이션

오류 처리
--------
- 입력 검증 실패/실행 예외는 messagebox로 사용자에게 즉시 표시
- 백그라운드 예외는 로그(traceback 포함) + 오류 다이얼로그 + busy 해제 순으로 처리

정리
----
- 이 UI는 "프로젝트 등록 → 계획 생성/수정 → 단계 실행 → closeout"의 전체 사이클을 단일 데스크톱 흐름으로 제공한다.
- 특히, pending 스텝 중심 편집과 실행 상태 시각화를 통해 체크포인트 기반 자동화를 안전하게 운영하는 데 초점을 둔다.

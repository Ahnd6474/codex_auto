codex-auto Desktop UI 기능 정리
=========================================

개요
----
- codex-auto의 현재 데스크톱 UI는 React + Tauri 기반 2-스테이지 화면 구조(설정 화면 + 실행 플로우 화면)로 동작한다.
- 목적은 여러 저장소를 워크스페이스 안에서 관리하고, Codex 기반 실행 계획 생성/편집/실행/closeout까지 하나의 데스크톱 흐름으로 처리하는 것이다.
- Python 오케스트레이션 백엔드는 유지되며, 데스크톱 UI는 `python -m codex_auto.ui_bridge`를 통해 연결된다.

실행
----
- 개발 실행
  - `cd desktop`
  - `npm.cmd install`
  - `npm.cmd run tauri:dev`
- 빌드
  - `cd desktop`
  - `npm.cmd run tauri:build`

워크스페이스/프로젝트 관리
--------------------------
- 관리 대상 프로젝트 목록을 카드 형태로 표시한다.
- 각 프로젝트는 표시 이름, 상태, 로컬 경로, 원격 URL 또는 브랜치 정보를 함께 보여준다.
- 새 프로젝트 생성, 기존 프로젝트 편집, 플로우 화면 열기, 목록 새로고침을 지원한다.
- 선택한 프로젝트의 요약 정보를 별도 패널에서 확인할 수 있다.

Setup Stage
-----------
- 프로젝트 설정 화면에서 다음 항목을 편집할 수 있다.
  - Project Name
  - Working Directory
  - Branch
  - Verification Command
  - Max Planned Steps
  - GitHub Connection Mode
  - Execution Model Preset
- GitHub 연결 방식은 기존 origin 사용, URL 직접 입력, 연결 안 함 세 가지를 지원한다.
- 저장만 하거나, 저장 후 바로 Flow Stage로 이동할 수 있다.

Flow Stage
----------
- Project Prompt를 입력하고 Codex로 실행 계획을 생성할 수 있다.
- 저장된 계획을 수정하고 다시 저장할 수 있다.
- 남은 스텝 실행, closeout 실행, 현재 스텝 종료 후 중단 요청을 지원한다.
- 플로우 스트립에서 각 스텝과 closeout 상태를 시각적으로 확인할 수 있다.
- pending 상태 스텝은 제목, 설명, Codex 지시문, 테스트 명령, 성공 기준을 편집할 수 있다.
- 스텝 추가, 삭제, 위/아래 이동, 선택 해제를 지원한다.

백그라운드 실행
---------------
- 긴 작업은 Tauri 백그라운드 잡으로 실행한다.
- 한 번에 하나의 잡만 허용해 상태 충돌을 줄인다.
- UI는 주기적으로 잡 상태를 폴링해 진행 상황과 결과를 반영한다.

추적성과 상태 표시
------------------
- Activity 탭에서 최근 UI 이벤트와 실행 흐름을 확인할 수 있다.
- Snapshot 탭에서 프로젝트, 런타임, 플랜, 최근 블록/패스, 사용량, 제어 상태를 JSON 형태로 볼 수 있다.
- stop-after-step 요청과 UI 이벤트 기록은 각 프로젝트 워크스페이스 파일로 남는다.

정리
----
- 현재 데스크톱 UI는 React + Tauri가 담당한다.
- Python 쪽은 GUI 프레젠테이션이 아니라 오케스트레이션, 상태 관리, 계획 실행, 보고, 브리지 처리에 집중한다.

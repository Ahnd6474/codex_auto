# Jakal-Flow UI 리디자인 분석 및 개선안

> **분석 관점**: 시니어 제품 디자이너 + UX 디자이너 + 디자인 시스템  
> **목표**: 첫 번째 UI 의 기본 구조 유지 + 두 번째 UI 의 강점 이식  
> **방향**: "IDE 형 탐색성" + "운영 중인 워크플로우 시스템의 상태 가독성" 결합

---

## 1. 한 줄 결론

**"첫 번째 UI 의 IDE 형 탐색 구조를 유지하면서, 두 번째 UI 의 '실시간 상태 가시성'과 '행 흐름 시각화'를 상단/우측 패널에 계층적으로 이식하라."**

---

## 2. 두 번째 UI 에서 반드시 가져와야 할 핵심 장점

| # | 장점 이름 | 왜 좋은지 | 첫 번째 UI 의 부족함 | 첫 번째 UI 적용 방법 |
|---|-----------|-----------|---------------------|---------------------|
| 1 | **LIVE RUN 배지** | 현재 실행 중임을 즉시 인지 | 정적 상태 표현 | 상단 바에 실행 상태 배지 추가 |
| 2 | **진행 상황 progress bar** | 전체 진행률을 한눈에 파악 | 텍스트 중심 (5/5 Done) | 상단/우측에 시각적 progress bar |
| 3 | **현재 단계 강조** | "지금 무엇을 하는가" 명확 | 모든 노드가 동일하게 표시 | 실행 중 노드를 강조 색상/크기 |
| 4 | **남은 시간 추정** | 작업 계획 수립 가능 | 없음 | 우측 패널에 ETA 표시 |
| 5 | **Reservations 시스템** | 다음 작업을 미리 인지 | 없음 | 우측 패널에 대기 큐 표시 |
| 6 | **단계별 상세 정보** | 각 단계의 목적/입출력 명확 | 노드 내 텍스트만 표시 | 우측에 단계 상세 카드 추가 |
| 7 | **상태별 색상 체계** | idle/running/completed 구분 | 초록색 중심 단조로움 | 상태별 색상 토 정의 |
| 8 | **Compact stepper** | 전체 흐름을 한눈에 | 복잡한 그래프 뷰 | 상단에 간단한 단계 stepper |
| 9 | **행 모델 명시** | 어떤 모델로 실행되는지 | 하단 상태바에만 표시 | 우측 패널에 모델 정보 카드 |
| 10 | **Deadlline/우선순위** | 작업 긴급도 인지 | 없음 | 프로젝트 카드에 데드라인 표시 |
| 11 | **Parallel Limit 표시** | 병렬 처리 한계 인지 | 설정에만 존재 | 우측 패널에 명시적 표시 |
| 12 | **Cancel Reservation** | 대기열 제어 가능 | 없음 | 액션 버튼으로 추가 |

## 3. 영역별 구체적인 화면 수정안

### 3.1 상단 바 (Top Bar)

#### 현재 문제
- 상태 정보가 텍스트 중심 (예: "Closed out > Completed 5/5 steps")
- 현재 행 여부를 즉시 판단하기 어려움
- 진행률이 숫자로만 표현됨

#### 두 번째 UI 에서 참고할 점
- "LIVE RUN" 배지로 실행 중임을 강조
- Progress bar 로 시각적 진행도 표시
- 현재 작업 단계를 텍스트로 명시

#### 구체적인 수정안

```
[기존]
[로고] [프로젝트 선택 드롭다운]          [상태 텍스트]           [Settings] [Run]

[개선안]
[로고] [프로젝트 선택] │ [LIVE RUN] Working on ST4 - Reconcile...  [====50%====] [Running 배지]
                       │ Completed 3/6 steps, running: ST4
```

**추가 요소:**
- 실행 중일 때만 보이는 **진행바** (상단 하단 edge 에 thin bar)
- 현재 단계 이름을 **truncate** 하여 표시
- 상태 배지 (Running/Queued/Idle/Closed Out) 를 색상 + 아이콘으로 구분

#### 기대 효과
- 화면을 여는 순간 **실행 상태** 인지
- **어느 단계** 진행 중인지
- **전체 진행률** 이 어느 정도인지 1 초 내 인지 가능

---

### 3.2 좌측 프로젝트/탐색 패널 (Left Explorer)

#### 현재 문제
- 프로젝트 상태가 텍스트 배지만 (예: "Closed Out")
- 실행 중인 프로젝트가 무엇인지 강조 부족
- 현재 선택된 프로젝트 외의 상태 변화 인지 어려움

#### 두 번째 UI 에서 참고할 점
- 프로젝트 카드에 **실행 상태 + 현재 단계** 동시 표시
- 실행 중인 프로젝트는 **배경색/테두리**로 강조
- Queued/Running/Completed 를 **색상 토큰**으로 구분

#### 구체적인 수정안

```
[기존]
┌────────────────────────────
│ jakal-math-generator       │
│ https://github.com/...     │
│ [Closed Out]               │
└────────────────────────────┘

[개선안]
┌────────────────────────────┐
│ ▶ jakal-work               │
│    [Running] ST4           │  ← 행 중 강조 (배경색/테두리)
│    ████████░░ 67%          │  ← 미니 progress bar
│    Est. 41 분 59 초          │
├────────────────────────────┤
│ experiment2                │
│ [Queued] Generate Plan     │  ← 대기 상태
────────────────────────────┤
│ lit                        │
│ [Plan Completed]           │
└────────────────────────────┘
```

**추가 요소:**
- 행 중인 프로젝트는 **왼쪽 테두리 accent color**
- 각 프로젝트 카드에 **미니 진행바** (sparkline 형태)
- **현재 단계 이름** 을 프로젝트 목록에서 직접 표시
- Hover 시 **퀵 액션** (Stop/Resume) 표시

#### 기대 효과
- 프로젝트 전환 시 **상태 맥락** 유지
- 다중 프로젝트 운영 시 **우선순위** 인지
- **방치된 실행** 방지

---

### 3.3 중앙 메인 작업 영역 (Main Work Area)

#### 현재 문제
- AI Chat 영역이 너무 넓고 **상태 정보**가 부족
- "Send a message..." 텍스트만 있고 **맥락**이 없음
- 행 중일 때 **피드백**이 약함

#### 두 번째 UI 에서 참고할 점
- 중앙은 **메시지/입력** 에 집중
- 상태 정보는 **상단/우측** 으로 분리
- 로딩/실행 중일 때 **스켈레톤/인디케이터**

#### 구체적인 수정안

```
[상단 탭]
[AI Chat] [Project Settings] [Dashboard] [History] [Reports]

[메인 영역 - 비어있을 때]
┌─────────────────────────────────────┐
│                                   │
│  저장된 txt 이력에서 세션을         │
│  이어가려면 메시지를 보내세요.      │
│                                     │
│  [현재 상태: Closed Out]            │  ← 맥락 정보 추가
│  마지막 실행: 2026-04-02 14:30      │
│  다음 단계: ST1 - Stabilize shell   │
└─────────────────────────────────────┘

[메인 영역 - 실행 중일 때]
┌─────────────────────────────────────┐
│  [ST4 진행 중...]                   │
│  ████████████░░░░░░ 67%             │
│  Est. 41 분 59 초 남음               │
│                                     │
│  > 코드 리뷰 중...                  │  ← 실시간 로그
│  > 파일 3 개 수정 완료              │
└─────────────────────────────────────┘
```

#### 기대 효과
- **맥락 있는** 빈 상태
- 실행 중 **실시간 피드백**
- **불안감 감소** (시스템이 작동 중임을 인지)

---

### 3.4 우측 실행/상태 패널 (Right Execution Panel)

#### 현재 문제
- 그래프 뷰가 **축소**되면 정보 가치가 떨어짐
- 상태 정보가 **분산** (상단에도, 그래프에도)
- **다음 행동**이 무엇인지 명확하지 않음

#### 두 번째 UI 에서 참고할 점
- **Stepper** 형태의 단계 나열
- **Reservations** 로 대기열 표시
- **상세 정보 카드** 로 현재 단계 설명
- **실행 모델** 정보 명시

#### 구체적인 수정안

```
┌─────────────────────────────────────┐
│ [Running: run plan]    [3/6 Done]   │
│ [1 Ready Nodes]        [1 Reservations] │
│                                     │
│ 41 분 59 초 Est. Remaining            │
│ 2 개 워커 Parallel Limit             │
│                                     │
│ [Generate] [Save] [Reset]           │
│ [Run ▼] [Stop]                      │
│ [ ] Auto-run After Plan             │
├─────────────────────────────────────┤
│ [STP] → [ST2] → [ST4] → [ST5]      │  ← Stepper
│  Done   Running  Next   Pending     │
├─────────────────────────────────────┤
│ Reservations (1) jakal-work #1      │
│ [Cancel Reservation]                │
├─────────────────────────────────────┤
│ CO1: Closeout                       │
│ ─────────────────────────────────── │
│ Status: [Pending]                   │
│ Progress: 6/6                       │
│ Deadline: 2026-04-05 (높음)         │
│ Model: Auto (AGENTS.md pr--)        │
│ Execution: Use execution model      │
└─────────────────────────────────────┘
```

**그래프 뷰 개선:**
- 그래프는 **접기/펼치기** 가능하게
- 축소 시 **stepper** 로 전환
- 확대 시 **상세 노드** 표시

#### 기대 효과
- **계층적 정보** (중요도 순)
- **다음 행동** 명확
- **대기열** 인지

---

### 3.5 하단 입력창/액션 영역 (Bottom Composer)

#### 현재 문제
- 입력창이 **고립**되어 있음
- 실행 모델 선택이 **드롭다운**으로 숨겨짐
- **전송 버튼** 외 액션 부족

#### 두 번째 UI 에서 참고할 점
- 입력창 우측에 **실행 버튼** 통합
- 모델 정보는 **좌측**에 명시
- **컨텍스트** 유지

#### 구체적인 수정안

```
┌─────────────────────────────────────────────────────┐
│ [+] [Default: code review]                          │
│ ─────────────────────────────────────────────────── │
│ 실행 모델: [프로젝트 실행 모델 ▼]                   │
│ REASONING: [Project default ▼]      │
│                                                     │
│ ┌─────────────────────────────────┐  [보내기]       │
│ │ 코드나 변경 내용을 입력하세요   │  (초록색 강조) │
│ │ (Enter 로 전송)                 │                │
│ ─────────────────────────────────┘                │
─────────────────────────────────────────────────────┘
```

**상태별 변경:**
- 행 중: 입력창 **disable**, "실행 완료 후 입력 가능합니다" 표시
- 대기 중: "계획 생성 후 실행 가능합니다" 표시
- **전송 버튼** 을 상태에 따라 **Run/Stop/Resume** 으로 변경

#### 기대 효과
- **명확한 액션**
- **상태 기반** 입력 제어
- **모델 정보** 노출

## 4. 정보 위계 재설계안

### 사용자가 화면을 열었을 때 정보 인식 순서

```
1 차 정보 (0.5 초 내 인지)
── 현재 실행 상태 (Running/Idle/Closed Out)
├── 현재 진행 중인 단계 (ST4 - Reconcile...)
└── 전체 진행률 (3/6, 50%)

2 차 정보 (2 초 내 인지)
├── 다음 단계 (ST5 - ...)
├── 예상 남은 시간 (41 분 59 초)
├── 대기열 (Reservations)
└── 병렬 처리 한계 (2 개 워커)

3 차 정보 (필요 시 확인)
├── 실행 모델 정보
├── 데드라인/우선순위
├── 과거 이력
└── 상세 로그
```

### 정보 우선순위 테이블

| 우선순위 | 정보 | 위치 | 표시 방식 |
|----------|------|------|-----------|
| P0 | 실행 상태 | 상단 바 | 배지 + 색상 |
| P0 | 현재 단계 | 상단 바 | 텍스트 |
| P0 | 진행률 | 상단 바 | Progress bar |
| P1 | 다음 단계 | 우측 패널 | Stepper |
| P1 | 예상 시간 | 우측 패널 | 텍스트 |
| P1 | 대기열 | 우측 패널 | 카드 |
| P2 | 실행 모델 | 우측 패널 하단 | 작은 텍스트 |
| P2 | 데드라인 | 프로젝트 카드 | 배지 |
| P3 | 상세 로그 | 접이식 패널 | 로그 뷰어 |

## 5. 상태 표현 체계 제안

### 상태별 컴포넌트/표현 방법

| 상태 | 색상 | 아이콘 | 컴포넌트 | 표시 위치 |
|------|------|--------|----------|-----------|
| **Idle** | Gray (#6B7280) | ⏸ | 배지 | 상단 바, 프로젝트 카드 |
| **Queued** | Blue (#3B82F6) | ⏳ | 배지 + 대기열 카드 | 상단 바, 좌측 패널 |
| **Running** | Green (#10B981) | ▶ | 배지 + Progress bar | 상단 바, 우측 패널 |
| **Paused** | Yellow (#F59E0B) |  | 배지 + Resume 버튼 | 상단 바, 우측 패널 |
| **Completed** | Green (#10B981) | ✓ | 배지 + 체크마크 | 프로젝트 카드, 그래프 |
| **Failed** | Red (#EF4444) | ⚠ | 배지 + 에러 메시지 | 상단 바, 우측 패널 |
| **Closed Out** | Purple (#8B5CF6) | ◎ | 배지 + 요약 카드 | 프로젝트 카드, 상단 바 |

### 상태별 UI 변화

```
[Idle]
- 입력창: 활성화
- Run 버튼: primary
- 그래프: 정적

[Running]
- 상단: LIVE RUN 배지 + 진행바
- 입력창: disable + "실행 중..." 메시지
- Run 버튼: Stop 으로 변경 (red)
- 그래프: 현재 노드 애니메이션 (펄스)

[Paused]
- 상단: PAUSED 배지
- 입력창: 제한적 활성화
- Run 버튼: Resume 으로 변경 (green)

[Failed]
- 상단: FAILED 배지 (red)
- 에러 메시지: 우측 패널 상단
- Retry 버튼: 표시

[Completed/Closed Out]
- 상단: COMPLETED 배지 (green)
- 입력창: 활성화 (다음 작업 가능)
- Run 버튼: Run (다음 단계)
```

## 6. 단계/워크플로우 시각화 개선안

### 현재 그래프 뷰 문제
- 노드가 **작아지면** 정보 가치 하락
- **진행 방향**이 직관적이지 않음
- **현재 단계** 강조 부족

### 개선안: Hybrid Stepper + Graph

```
[상단 Stepper - 항상 표시]
┌─────────────────────────────────────────────────────┐
│ [ST1] ──→ [ST2] ──→ [ST4] ──→ [ST5] ──→ [CO1]      │
│  Done     Running    Next     Pending   Pending     │
│  ✓        ▶          ·         ·         ·          │
└─────────────────────────────────────────────────────┘

[우측 패널 - 상세 뷰 (펼치기 가능)]
─────────────────────────────────────────────────────┐
│                     그래프 뷰                       │
│                                                     │
│        ┌─────────┐                                  │
│        │   ST1   │                                  │
│        │  Done   │                                  │
│        └────┬────┘                                  │
│             │                                       │
│             ▼                                       │
│        ┌─────────┐     ┌─────────┐                 │
│        │   ST2   │ ──→ │   ST3   │                 │
│        │ Running │     │ Pending │                 │
│        └────┬────┘     └─────────┘                 │
│             │                                       │
│             ▼                                       │
│        ┌─────────┐                                  │
│        │   ST4   │                                  │
│        │  Next   │                                  │
│        └─────────┘                                  │
─────────────────────────────────────────────────────┘
```

### 진행률 표시

| 컴포넌트 | 사용처 | 설명 |
|----------|--------|------|
| **Linear Progress Bar** | 상단 바 | 전체 진행률 (3/6 = 50%) |
| **Circular Progress** | 우측 패널 | 현재 단계 진행률 |
| **Stepper** | 상단/우측 | 단계별 상태 (Done/Running/Next) |
| **노드 애니메이션** | 그래프 | 실행 중 노드 스 효과 |
| **텍스트** | 모든 영역 | "3/6 Done", "50%" |

### 사용자가 즉시 이해해야 할 정보

1. **현재 어느 단계인지** → Stepper 에서 **현재 노드 강조** (색상 + 크기 + 아이콘)
2. **이전 단계는 끝났는지** → Done 노드는 **초록색 + 체크마크**
3. **다음 단계는 무엇인지** → Next 노드는 **회색 + 점선 테두리**
4. **전체 진행률** → 상단 **Progress bar** + 텍스트 (50%)

## 7. 액션 버튼 체계 개선안

### 버튼 우선순위 재정리

| 우선순위 | 버튼 | 유형 | 상태 | 설명 |
|----------|------|------|------|------|
| **Primary** | Run | Primary | 실행 가능 시 활성화 | 워크플로우 실행 |
| **Primary** | Stop | Danger | 행 중일 때만 | 현재 실행 중지 |
| Secondary | Generate | Secondary | 계획 생성 전 | 계획 생성 |
| Secondary | Save | Secondary | 항상 | 상태 저장 |
| Secondary | Reset | Secondary | 항상 | 상태 초기화 |
| Secondary | Resume | Primary | Pause 중 | 실행 재개 |
| Tertiary | Cancel Reservation | Tertiary | 대기열 있을 때 | 대기열 취소 |

### 상태별 버튼 변화

| 현재 상태 | 표시될 버튼 | 비활성화 | 숨김 |
|-----------|-------------|----------|------|
| **Idle** | Run, Generate, Save, Reset | - | Stop |
| **Running** | Stop, Save | Run, Generate | - |
| **Paused** | Resume, Stop, Save | Run, Generate | - |
| **Queued** | Cancel, Save | Run, Stop | - |
| **Completed** | Run (다음), Save, Reset | - | Stop |
| **Failed** | Retry, Save, Reset | Run | Stop |
| **Closed Out** | Run (새 작업), Save | - | Stop |

### 버튼 레이블 동적 변경

```
[Idle]        → [Run]
[Running]     → [Stop]
[Paused]      → [Resume]
[Queued]      → [Cancel]
[Failed]      → [Retry]
[Completed]   → [Run Next]
```

## 8. 첫 번째 UI 에 유지해야 할 장점

### 절대 버리면 안 되는 요소

| 요소 | 왜 중요한가 | 유지 방법 |
|------|-------------|-----------|
| **IDE 형 탐색 패널** | 파일/프로젝트 구조 파악 | 좌측 패널 유지, 정보만 추가 |
| **프로젝트 목록** | 다중 프로젝트 관리 | 상태 정보만 추가, 구조 유지 |
| **파일 트리** | 세부 파일 접근 | 하단에 유지, 접기/펼치기 |
| **탭 기반 네비게이션** | 기능 전환 | AI Chat/Settings/Dashboard 유지 |
| **입력창 구조** | 코드 리뷰/명령 입력 | 레이아웃 유지, 상태 정보 추가 |
| **하단 상태바** | 글로벌 상태 | 정보 밀도 조절, 유지 |
| **그래프 뷰** | 워크플로우 구조 | Stepper 와 hybrid 으로 |

### "유지할 것" vs "바꿀 것"

| 유지할 것 | 바꿀 것 |
|-----------|---------|
| 좌측 탐색 패널 구조 | 프로젝트 카드 정보 (상태/진행) |
| 중앙 AI Chat 영역 | 빈 상태 메시지 (맥락 추가) |
| 우측 패널 위치 | 그래프 → Stepper hybrid |
| 하단 입력창 구조 | 행 버튼 통합 |
| 탭 기반 네비게이션 | 상단 바 정보 밀도 |
| 파일 트리 | 상태 배지 체계 |

## 11. 가장 추천하는 최종 방향

> **"상단 바에 '행 상태 + 진행률 + 현재 단계'를 집중 배치하고, 좌측 프로젝트 목록에는 '상태 배지 + 미니 진행바'만 추가하라. 우측 패널은 그래프 위에 'Stepper'를 추가하여 축퇴 시에도 흐름을 파악하게 하라."**

### 핵심 원칙

1. **0.5 초 규칙**: 화면을 연 지 0.5 초 내에 실 상태/진행률/현재 단계를 인지해야 한다.
2. **계층적 정보**: P0 정보 (상태) 는 항상, P1 정보 (예상 시간) 는 공간 여유 시, P2 정보 (모델) 는 요청 시.
3. **유지 + 강화**: 첫 번째 UI 의 탐색 구조를 유지하면서, 두 번째 UI 의 상태 가시성만 이식한다.
4. **상태 기반 UI**: 버튼/입력창/메시지를 현재 상태에 따라 동적으로 변경한다.

### 다음 액션

1. P0 항목부터 순차적 구현
2. 각 항목 완료 후 **사용자 테스트** (상태 인지 시간 측정)
3. P1/P2 는 리소스 여유에 따라 우선순위 조정
4. **A/B 테스트**: 기존 UI vs 개선 UI (작업 완료 시간/에러율 비교)

Workflow

사용자는 방향을 주고, Codex는 블록으로 쪼개고, 시스템은 엄격하게 검증합니다.

장기 프롬프트 또는 저장소를 입력으로 받아, Codex가 실행 가능한 작업 목록을 만들고 각 블록이 끝날 때까지 반복 수행합니다. 실패한 pass는 즉시 되돌리고, 성공한 변경만 안전 revision으로 남깁니다.

01
입력 확보

장기 프롬프트나 현재 저장소 상태를 읽습니다.

02
Codex 작업 분해

실행 가능한 블록과 우선순위를 생성합니다.

03
구현과 테스트

각 pass 뒤 테스트를 돌리고, 회귀 시 rollback 합니다.

04
체크포인트와 기록

상태, 로그, 메모리, 보고서를 남기고 GitHub와 동기화합니다.

Core Value

앱으로 발전시키기 쉬운 구조

운영 추적성

LONG_TERM_PLAN, MID_TERM_PLAN, ACTIVE_TASK, BLOCK_REVIEW가 남습니다.

안전한 실행 루프

테스트 실패 시 즉시 rollback 하고 safe revision만 유지합니다.

멀티 저장소 설계

repo, docs, logs, memory, state가 저장소별로 분리됩니다.

GUI 중심 확장성

지금은 데스크톱 GUI, 이후에는 배포용 앱 UX로 옮기기 좋습니다.

Release Readiness

배포 전에도 필요한 정보가 정리되어 있습니다

Workspace repo / docs / memory / logs / state / reports
Execution 모델 선택, 추론 강도, 블록 수, 테스트 명령 제어
Review 체크포인트 승인, 상태 요약, 진행 타임라인 제공

Positioning

자동화 툴이 아니라,
릴리즈 가능한 운영 레이어.

계획

장기 프롬프트와 저장소 문맥을 바탕으로 작업을 세분화합니다.

실행

Codex가 블록 단위로 구현을 진행하고 진행 상황을 남깁니다.

검증

매 pass 뒤 테스트하고 실패 시 즉시 rollback 합니다.

기록

docs, logs, memory, state를 남겨 이후 릴리즈 설명이 가능합니다.

Workflow

사람은 방향을 주고,
Codex는 블록으로 움직입니다.

01 Input

장기 프롬프트 또는 개선할 저장소를 입력으로 받습니다.

02 Codex Breakdown

Codex가 실제 구현 가능한 블록 목록과 우선순위를 제안합니다.

03 Execution Loop

블록이 끝날 때까지 구현, 테스트, 기록을 반복합니다.

04 Checkpoint

승인 경계와 GitHub 동기화를 통해 릴리즈 흐름을 유지합니다.

Why It Feels Different

제품처럼 보이고,
운영 도구처럼 작동합니다.

멀티 저장소 구조

repo, docs, memory, logs, reports, state를 저장소별로 분리해 섞이지 않게 관리합니다.

엄격한 검증

테스트 실패 시 safe revision으로 바로 되돌아갑니다.

진행 타임라인

현재 블록, 남은 흐름, Codex 상태를 직관적으로 보여줍니다.

앱 확장 전제

데스크톱 GUI에서 시작해 릴리즈 가능한 제품 경험으로 확장하기 쉽습니다.
